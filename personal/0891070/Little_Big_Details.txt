1. Github

commit 할 때 제목에 쓸 수 있는 문자의 개수가 50개로 제한되어 있는데 

50개가 넘어가는 문자들은 회색으로 처리하고 경고 메시지를 띄어준다.

예전에 싸이월드 할 때 방명록 이였나? 잘 기억은 안 나는데 글을 엄청 길게 쓰고 있었고

글을 남기려고 하니까 최대 쓸 수 있는 문자 길이를 넘어 갔다고 모든 내용들이 다 

삭제 된 일이 있었는데 이렇게 미리 제한 된 문자 길이를 초과했다고 알려주는 것은

사용자들의 스트레스를 줄이는 좋은 방법이라고 생각한다.

2. Google 날씨

구글 날씨에서 바람을 화살표로 표시하여 바람이 부는 방향을 가리키는데 이 때

화살표로 방향뿐만 아니라 화살표 사이즈도 바람이 세질수록 더 크게 보여준다.

디자인 측면에서 보기 좋고 한 눈에 정보를 볼 수 있어서 편리하다.

3. WordPress

로그인 할 때 이메일의 도메인 네임에 오타를 발견할 시 혹시 이 도메인 네임이 아니야?

라고 물어본다. 사용자와 대화하는 형식으로 추천을 해주는 게 친화력이 있는 아이디어

인 것 같다.

4. Gmail

수신자를 써야할 곳에 만약에 team at angel dot co 라고 치면 이것을 team@angel.co로

자동으로 전환해준다. 이 기능은 모바일에서 아주 편리하게 쓰일 것 같다. 모바일에서는

이 메일을 쓸 때 '@'나 '.'을 입력하기 위해서 기호 모드로 들어가야 하고 다시 text 모드로

돌아와야 하는 귀찮음을 제거해 줄 수 있다.

5. Amazon

쇼핑을 할 때 윈도우 창이 작아지면 전체 콘텐츠를 다 보여주는 것이 아니라 일부를 숨겨서

보여준다. 윈도우 창을 줄였을 때 비교적 사용 빈도가 낮거나 사용자의 관심 밖의 콘텐츠를 

숨김으로써 나머지 중요한 콘텐츠를 사용자가 효율적으로 볼 수 있게 된다.

6. Sleepbot

사용자가 어플을 사용하다가 잠 잘 시간이 돼도 어플을 종료하지 않으면 어플을 종료하도록

안내한다. 메모리와 배터리를 효율적으로 사용 할 수 있도록 도와준다.

7. iOS7

시계 어플 아이콘이 실제 시간과 똑같이 움직인다. 

어플을 실행하지 않아도 시간을 확인 할 수 있어서 사용자는 빨리 시간을 확인할 수 있고

실행을 해야 하는 번거로움을 없애준다.

8. Dropbox

좋은 비밀번호를 만들기 위한 팁에서 흔하지 않고 평범하지 않은 단어들을 쓰라고 재미있게

표현하고 있다. 예를 들어 uppercasing을 uPPercasing 이라고 적어서 대문자와 소문자를 

혼합하는 형태나 spelling을 spellllling처럼 l을 여러 개 적는 방법 등을 추천해 준다.

확실히 이렇게 비밀번호를 조합하면 보안성이 높아질 것이다.

9. Netflix

영상을 볼 때 마우스 커서를 영상 위에서 움직이면 재생과 재생위치를 보여주는 bar 같은 

컨트롤러가 생기는데 움직임이 없으면 바로 이 컨트롤러를 숨겨준다. 나도 영상을 자주 보는

편인데 컨트롤러가 올라오면 속으로 '빨리 없어져라!' 라고 생각하는 경우가 많았다. 

이 답답함을 속 시원하게 풀어주는 서비스인 것 같다!

10. iTunes

앨범 리스트를 볼 때 사용자가 선택한 앨범의 전체적인 색깔과 그 앨범의 리스트를 보여주는 

테이블의 색깔을 비슷하게 해준다. 역시 디자인을 생각하는 애플을 볼 수 있고 사용자는 

앨범 리스트를 앨범 전체의 디자인으로 보게 할 수 있어서 세련돼 보인다.
