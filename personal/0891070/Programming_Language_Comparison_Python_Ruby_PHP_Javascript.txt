1. Python
1) 먼저 파이썬은 우리가 평소에 다른 사람들과 대화 방식으로 소통을 하는 것처럼 인간의 언어와 가까운 문법을 가진 언어이다. 그래서 파이썬을 사용하면 컴퓨터와 소통하는 것에 있어서 아주 자연스럽게 프로그래밍을 할 수 있다. 또한 문법 자체가 인간의 언어와 비슷하다보니 파이썬을 공부하는 학생들 또한 쉽게 배우고 빠르게 프로그래밍을 할 수 있어서 전문가가 아니여도 개발을 쉽게 할 수 있다.
2) 파이썬으로 거의 모든 종류의 프로그램 개발이 가능하다. 물론 운영체제 프로그래밍 같이 복잡한 프로그래밍은 파이썬만으로는 어렵지만 빠른 속도를 자랑하는 C와 결합하여 사용하면 이 단점을 해결 할 수가 있다. 파이썬에서는 다른 언어의 모듈을 포함 할 수 있는데 이렇게 해서 빠른 속도를 필요로 하는 부분은 C로 프로그래밍을 하고 전체적인 틀을 파이썬으로 프로그래밍하여 C를 파이썬에 포함 시키는 것이다. 
3) 파이썬은 무료로 이용할 수 있으며 오픈소스이다.
4) 가독성이 좋다. 그러므로 소프트웨어 개발을 할 때뿐만 아니라 다른 개발자가 자신의 코드를 볼 경우에도 다른 언어에 비해 가독성이 좋아서 한 눈에 코드를 이해 할 수 있다는 것이다. 그리고 파이썬을 실행하기 위해서는 코드의 정해진 줄을 잘 맞춰야 한다. 가독성이 좋게 하기 위해서 코드들의 줄을 잘 맞춰야 할 필요가 있으나 파이썬의 경우 줄을 맞추지 않으면 실행이 되지 않아서 파이썬 개발자라면 누구든지 다른 사람의 코드를 잘 이해 할 수가 있고 이러한 성향은 팀 프로젝트에 많은 도움이 된다.
5) 개발 속도가 빠르다. 위의 장점들과 일맥상통하는 말이지만 이런 파이썬들의 장점들로 인해서 개발의 속도가 빨라진다. 파이썬을 처음 접하는 사람들도 하루 만에 간단한 프로그램을 개발 할 수 있다고 한다. 개발 속도라는 것이 소프트웨어 시장에서 아주 중요한 요소임은 틀림없기 때문에 파이썬의 중요성은 높아지고 있다.

2. Ruby
1) 배우기 쉽다. 루비를 만든 사람이 루비를 만들 때부터 배우기 쉬운 언어 지향적으로 개발을 했다고 한다. 그래서 루비를 마스터 하지 않아도 어느 정도 기본 문법으로 활용이 좋은 프로그램 개발이 가능하다.
2) 코드가 간결하다. 자바로 프로그래밍을 할 경우 코드가 500줄이라면 ruby on rails를 이용하면 50줄로 줄일 수 있다고 한다.
3) 개발 속도가 빠르다. ruby on rails를 이용하면 빠른 웹서비스 개발을 할 수 있다.
4) 정형화 되지 않은 자유로운 문법. 개발자는 직관적으로 프로그래밍을 할 수 있어서 편하고 가독성이 좋다.
5) 프로그래밍이 재미있다. 문법이 자유로운 편이기 때문에 개발자는 자신이 원하는 데로 재미있게 프로그래밍이 가능하다.

3. PHP
1) PHP는 빠르게 개발 할 수 있는 장점이 있다. 문법이 간단하게 표현되어 코딩하게 되어 있고 직관적으로 프로그래밍이 가능하여 편리하고 장기적으로 프로젝트를 할 때 빠르게 개발이 가능하다. 
2) 유지비용이 저렴하다. PHP는 무료 라이선스인 리눅스와 데이터베이스는 Mysql을 쓰는 경우가 많아서 다른 언어들의 비해서 아주 저렴한 편이다.
3) 문법 또한 간결한 편이다. PHP는 C와 C++언어의 문법과 비슷해서 클래스를 지원하고 스크립트 언어라서 HTML 내부에 포함 시킬 수 있다.
4) Mysql과의 호환성이 좋다. sql문을 쓸 때 간단하게 쓸 수 있고 cmd 없이도 직접 데이터베이스를 구동할 수 있다. 그래서 데이터베이스를 빠르게 처리 할 수 있다.
5) 객체 지향 프로그래밍이 가능하여 조금 더 복잡한 프로그래밍을 할 수 있다.

4. Javascript
1) 작업 속도가 빠르다. 자바스크립트를 실행시키기 위해서 다른 프로그램이 필요한 게 아니라 텍스트 문서와 같은 곳에서 바로 테스트 할 수 있고 웹브라우저에서 실행된다.
2) 운영 체제의 영향을 받지 않는다. 자바스크립트는 HTML코드 내에서 바로 실행이 됨으로 운영체제에 상관없이 개발이 가능하여 특정 작업 환경의 구애를 받지 않는다.
3) 배우기 쉽다. 자바스크립트는 다른 언어에 비해 문법이 쉽고 객체가 간소화되어 있어서 초보자도 빠르게 프로그램을 만들 수 있다.
4) 다른 웹 페이지를 열거나 서버 응답 시간을 기다리지 않고도 동적인 기능을 수행 할 수 있다.
5) 자바스크립트는 HTML 코드 내에 포함된 상태에서 전달돼서 추가적으로 다른 네트워크가 필요 없다.
